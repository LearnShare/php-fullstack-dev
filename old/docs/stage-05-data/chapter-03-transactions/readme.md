# 5.3 MySQL 事务处理

## 目标

- 理解 ACID 四特性的含义与重要性。
- 掌握事务的基本操作：`beginTransaction()`、`commit()`、`rollBack()`。
- 熟悉 MySQL 事务隔离级别及其影响。
- 了解死锁的产生原因与重试策略。

## 章节内容

本章分为三个独立小节，每节提供详细的概念解释、代码示例和最佳实践：

1. **[事务基础](section-01-transaction-basics.md)**：什么是事务、事务基本操作（beginTransaction、commit、rollBack）、事务示例、完整示例。

2. **[ACID 特性](section-02-acid.md)**：ACID 四特性详解（原子性、一致性、隔离性、持久性）、ACID 示例、完整示例。

3. **[隔离级别](section-03-isolation-levels.md)**：事务隔离级别、脏读、不可重复读、幻读、隔离级别设置、死锁处理及完整示例。

## 核心概念

- **事务**：一组数据库操作，要么全部成功，要么全部失败
- **ACID**：原子性、一致性、隔离性、持久性
- **隔离级别**：控制并发事务的隔离程度
- **死锁**：多个事务相互等待资源
