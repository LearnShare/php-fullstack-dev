# 2.17 常见错误与调试技巧

## 目标

- 识别和解决常见的 PHP 错误，掌握系统化排查思路。
- 运用调试工具与日志手段快速定位问题，降低定位成本。
- 建立性能排查与质量保障流程，形成自检清单与实践套路。

## 章节内容

本章拆分为三个独立小节，按“发现问题 → 定位问题 → 建立治理流程”的顺序展开：

1. **[常见错误类型与修复流程](section-01-common-errors.md)**：覆盖语法、类型、变量、内存等典型错误，结合示例给出快速定位与修复步骤。
2. **[调试技巧与工具链](section-02-debugging-techniques.md)**：总结错误报告、异常/错误处理、日志记录、性能分析与数据库调试等通用手段，并演示 Xdebug、VarDumper、Whoops 等工具。
3. **[排查清单与最佳实践](section-03-troubleshooting-checklist.md)**：提供常见问题自检列表、调试流程化建议与练习任务，帮助团队建立可重复的质量保障体系。

## 使用建议

- 结合日常开发中遇到的真实错误，对照第一节示例逐项复盘，记录排查路径。
- 在项目模板中固化日志、异常、性能监控三件套，确保“可观测”。
- 建立团队共享的排查清单与 SOP，输出每次故障的复盘与改进项。
