# 3.2 现代 PHP 8+ OOP 能力

## 目标

- 掌握构造器属性提升（Constructor Property Promotion）语法，减少重复代码。
- 理解 `readonly` 属性的作用与使用场景，创建不可变对象。
- 熟悉枚举（Enums）的定义与使用，替代传统常量类。
- 了解属性钩子（Property Hooks）和静态方法的合理使用。

## 章节内容

本章分为四个独立小节，每节提供详细的概念解释、语法说明、参数列表和完整示例：

1. **[构造器属性提升](section-01-constructor-promotion.md)**：基础语法、可见性修饰符、混合使用、默认值、类型声明及完整示例。

2. **[Readonly 属性](section-02-readonly.md)**：Readonly 属性基础、Readonly 类、与克隆的关系、最佳实践（值对象、DTO、配置对象）及完整示例。

3. **[枚举（Enums）](section-03-enums.md)**：基础枚举、Backed Enums、枚举方法、接口实现、与 match 表达式配合、数据库存储及完整示例。

4. **[属性钩子与静态方法](section-04-property-hooks.md)**：属性钩子基础语法、不对称可见性、静态属性滥用风险、静态方法合理使用、静态工厂方法及完整示例。

## 核心特性

- **构造器属性提升**：减少样板代码，提升可读性
- **Readonly**：创建不可变对象，提高安全性
- **枚举**：类型安全的常量集合
- **属性钩子**：自定义属性访问逻辑

## 学习建议

1. **按顺序学习**：按顺序学习四个小节，理解现代 PHP OOP 特性。

2. **重点掌握**：
   - 构造器属性提升的使用
   - Readonly 属性和类的应用
   - 枚举的定义和使用
   - 属性钩子的语法和应用

3. **实践练习**：
   - 完成每小节后的练习题目
   - 使用现代特性重构现有代码
   - 理解不可变对象的设计

## 完成本章后

- 能够使用构造器属性提升简化代码。
- 能够使用 readonly 创建不可变对象。
- 能够定义和使用枚举替代传统常量。
- 能够使用属性钩子（PHP 8.4+）自定义属性访问。
