# 4.7 响应处理与跨域（CORS）

## 目标

- 掌握 HTTP 状态码的设置方法与常用状态码。
- 熟悉响应头的设置与常用 Header。
- 深入理解 CORS（跨域资源共享）机制。
- 能够正确处理预检请求（Preflight）和跨域问题。

## 章节内容

本章分为两个独立小节，每节提供详细的概念解释、代码示例和最佳实践：

1. **[响应处理](section-01-response-handling.md)**：HTTP 状态码设置、响应头设置、常用响应头、响应格式、错误响应及完整示例。

2. **[CORS 跨域处理](section-02-cors.md)**：CORS 机制、预检请求（Preflight）、CORS 配置、安全最佳实践、完整示例及常见问题解决。

## 核心概念

- **HTTP 状态码**：表示请求处理结果
- **响应头**：控制响应行为
- **CORS**：跨域资源共享机制
- **预检请求**：OPTIONS 请求，用于 CORS 验证

## 学习建议

1. **重点掌握**：
   - HTTP 状态码的正确使用
   - CORS 的配置和处理
   - 预检请求的处理

2. **实践练习**：
   - 完成每小节后的练习题目
   - 实现 CORS 中间件
   - 处理各种跨域场景

## 完成本章后

- 能够正确设置 HTTP 状态码和响应头。
- 理解 CORS 机制，能够处理跨域请求。
- 掌握预检请求的处理方法。
- 具备构建生产级 API 响应的能力。
