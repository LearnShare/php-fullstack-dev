# 文档内容建立规则总结

本文档总结了 PHP + MySQL 全栈开发指南的文档建立规则和风格，所有文档必须遵循这些规则。

## 一、编号规则

### 基本规则

- **所有编号从 1 开始，不使用 0**
- **遵循 1.2.3 这种结构**（阶段.章节.小节）
- **示例**：
  - ✅ 正确：1.1、1.2、1.3
  - ❌ 错误：1.0、1.0.1、1.0.2

### 编号层级

- **阶段编号**：1、2、3...（阶段一、阶段二、阶段三）
- **章节编号**：1.1、1.2、1.3...（阶段一的第1、2、3章）
- **小节编号**：1.1.1、1.1.2、1.1.3...（第1章的第1、2、3节）

## 二、文档创建顺序

### 步骤 1：创建完整目录结构

- 创建所有阶段、章节、小节的目录结构
- 目录命名：小写字母、数字、连字符（-）
- 示例：`docs/stage-01-foundation/chapter-01-history/section-01-history.md`

### 步骤 2：完成阶段总览文档

- 文件：`docs/stage-XX-xxx/readme.md`
- 包含内容：定位、前置知识、学习时间、练习建议、章节列表、完成能力、相关章节

### 步骤 3：完成章节总览文档

- 文件：`docs/stage-XX-xxx/chapter-XX-xxx/readme.md`
- 包含内容：目标、章节内容、核心概念、学习建议、完成能力、相关章节

### 步骤 4：完成小节结构标题和大纲

- 文件：`docs/stage-XX-xxx/chapter-XX-xxx/section-XX-xxx.md`
- **不填充完整内容**，只创建结构标题和详细大纲
- 包含：概述（含章节类型）、详细内容大纲、必需要素说明

## 三、阶段总览文档结构（readme.md）

### 必需部分

1. **阶段标题**：`# 阶段X：标题（英文名）`
2. **阶段描述**：简要说明本阶段的作用
3. **定位**：一句话说明本阶段的定位
4. **前置知识要求**：
   - 必须掌握
   - 不需要掌握
   - 如何检查（检查点列表）
5. **学习时间估算**：
   - 建议学习时间
   - 时间分配建议
6. **练习建议**：
   - 基础练习（每章完成后）
   - 综合练习（阶段完成后）
7. **章节内容**：
   - 列出所有章节，包含链接和简要说明
   - 列出每个章节的所有小节
8. **完成本阶段后**：列出完成本阶段后应具备的能力
9. **相关章节**：列出相关阶段和章节

## 四、章节总览文档结构（readme.md）

### 必需部分

1. **章节标题**：`# X.X 章节标题`
2. **目标**：列出学习目标（3-5 条）
3. **章节内容**：
   - 说明本章分为几个小节
   - 列出每个小节的标题、链接和简要说明
4. **核心概念/特性/概览**：
   - 根据章节类型选择合适的内容
   - 概念性章节：核心概念
   - 配置性章节：配置概览
   - 工具性章节：工具链概览
5. **学习建议**：
   - 按顺序学习
   - 重点掌握
   - 实践练习
6. **完成本章后**：列出完成本章后应具备的能力
7. **相关章节**：列出相关章节

## 五、小节文档结构（section-XX-xxx.md）

### 概述部分（必需）

```markdown
## 概述

（简要说明本节内容）

**章节类型**：概念性章节 / 配置性章节 / 语法性章节 / 工具性章节 / 实践性章节 / 参考性章节 / 最佳实践章节 / 对比性章节

**主要内容**：
- 内容点 1
- 内容点 2
- 内容点 3
```

### 根据章节类型确定必需要素

#### 概念性章节（Conceptual）

**必需要素**：
- 概述
- 核心内容（详细大纲）
- 总结

**可选要素**：
- 特性
- 常见问题
- 相关资源

**示例**：1.1.1 PHP 起源与发展历程

#### 配置性章节（Configuration）

**必需要素**：
- 概述
- 特性
- 配置步骤/语法
- 注意事项
- 常见问题

**可选要素**：
- 输出结果说明
- 对比分析
- 最佳实践

**示例**：1.2.1 PHP 安装与版本管理

#### 工具性章节（Tool）

**必需要素**：
- 概述
- 特性
- 基本用法/命令
- 使用场景
- 注意事项
- 常见问题
- 最佳实践

**可选要素**：
- 输出结果说明
- 对比分析
- 参数详情

**示例**：1.4.1 Composer 基础

#### 语法性章节（Syntax）

**必需要素**：
- 概述
- 特性
- 语法/定义
- 基本用法
- 代码示例
- 使用场景
- 注意事项
- 最佳实践

**可选要素**：
- 参数详情
- 返回值/状态
- 输出结果说明
- 对比分析
- 常见问题

#### 实践性章节（Practical）

**必需要素**：
- 概述
- 实践步骤
- 完整代码示例
- 输出结果说明
- 注意事项
- 练习任务

**可选要素**：
- 特性
- 常见问题
- 最佳实践

#### 参考性章节（Reference）

**必需要素**：
- 概述
- 定义/语法
- 示例
- 使用场景

**可选要素**：
- 特性
- 参数详情
- 返回值/状态
- 对比分析
- 注意事项

#### 最佳实践章节（Best Practices）

**必需要素**：
- 概述
- 实践建议/原则
- 代码示例
- 最佳实践
- 注意事项

**可选要素**：
- 特性
- 对比分析
- 常见问题
- 输出结果说明

#### 对比性章节（Comparative）

**必需要素**：
- 概述
- 对比分析（核心内容）
- 使用建议
- 注意事项

**可选要素**：
- 特性
- 代码示例
- 常见问题
- 最佳实践

### 详细内容大纲

每个小节必须包含**详细的内容大纲**，列出主要章节标题和内容说明，例如：

```markdown
## 核心内容

### 主要章节标题 1
- 子内容点 1
- 子内容点 2
- 子内容点 3

### 主要章节标题 2
- 子内容点 1
- 子内容点 2
```

## 六、内容编写规范

### 语言要求

- 使用简体中文
- 客观、严谨、简洁
- 禁止使用情感化修饰语
- 禁止使用非技术性比喻
- 禁止使用营销类词汇

### 详细程度

- **目标**：充分详实、零基础可学
- **行数参考**：300-500 行作为详实度范例
- 所有概念都要有充分的解释
- 所有语法都要有完整说明
- 所有代码示例都要完整可运行

### 代码示例

- 必须使用 PHP 8.2+ 语法
- 必须符合 PSR-12 编码规范
- 必须包含类型声明
- 必须完整可运行
- 使用标准 markdown 代码块（带语言标签）

### 表格对齐

- 表格必须在源文件中按 `|` 对齐列宽
- 使用对齐标记（`|:---|`）
- 宽度比例：9 中文 ≈ 15 英文

## 七、文件命名规范

- **所有文件**：小写字母、数字、连字符（-）
- **统一使用**：`readme.md`（小写），不使用 `README.md`
- **目录名**：同样遵循小写字母、数字、连字符（-）的规范

## 八、禁止事项

- ❌ 禁止使用脚本批量操作
- ❌ 禁止使用 CODE REFERENCES 格式
- ❌ 禁止编号从 0 开始
- ❌ 禁止在概述中使用"（待填充）"等占位符
- ❌ 禁止在小节中只写"（待填充）"而不提供大纲

## 九、工作流程

1. **创建目录结构**：逐个创建，不使用脚本
2. **完成阶段总览**：所有阶段的总览文档
3. **完成章节总览**：所有章节的总览文档
4. **完成小节大纲**：所有小节的结构标题和详细大纲
5. **内容填充**：在审核通过后，再填充完整内容
