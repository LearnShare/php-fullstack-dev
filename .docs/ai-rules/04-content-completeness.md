# 内容完整性要求（核心规则）

## 受众定位

- **所有文档面向零基础学员撰写**
- 语言需循序渐进但内容必须详实完整
- 避免过度简化，确保零基础学员能够完全理解
- 每个概念都要有充分的解释和多个示例

## 内容详细度

**目标**：充分详实、零基础可学

**行数参考**：300-500 行作为详实度范例，视主题复杂度可增减

**必须包含的要素**：
1. 详细概述
2. 特性说明
3. 多种创建/使用方式
4. 完整的语法说明
5. 参数列表（见方法/函数说明规范）
6. 返回值说明（见方法/函数说明规范）
7. 多个完整示例
8. 输出结果说明
9. 注意事项
10. 常见问题
11. 最佳实践
12. 练习任务（3-5 个）

## 方法/函数/API 说明规范

**介绍任何内置方法、函数、模块时，必须包含**：

1. **语法格式**：
   ```php
   functionName($param1, $param2 = 'default')
   ```

2. **参数列表**（表格格式）：
   | 参数名      | 类型   | 说明           | 是否必需 | 默认值 |
   |:------------|:-------|:---------------|:---------|:-------|
   | `$param1`   | string | 参数说明       | 是       | -      |
   | `$param2`   | string | 可选参数说明   | 否       | 'default' |

3. **返回值说明**：
   - 返回类型
   - 返回值含义
   - 特殊情况说明

4. **完整示例代码**（可运行）：
   ```php
   $result = functionName('value', 'option');
   var_dump($result); // 输出说明
   ```

5. **输出结果说明**：
   - 明确说明代码执行后的输出
   - 解释输出含义

6. **注意事项**：
   - 使用限制
   - 性能考虑
   - 兼容性说明

7. **常见错误**：
   - 常见陷阱
   - 错误示例及修正方法

## 对比说明

- 提供与其他数据结构、方法的对比
- 说明使用场景和选择建议
- 示例：`array_map()` vs `array_walk()` vs `foreach`

## 练习任务

**每个章节必须包含 3-5 个练习任务**：
- 练习应覆盖章节核心知识点
- 从简单到复杂，循序渐进
- 提供明确的练习目标
- 示例：
  ```markdown
  ## 练习

  1. 使用 `array_map()` 方法将数组 `[1, 2, 3]` 中的每个元素乘以 2。
  2. 使用 `array_filter()` 方法筛选出数组中的偶数。
  3. 使用 `array_reduce()` 方法计算数组元素的总和。
  ```
