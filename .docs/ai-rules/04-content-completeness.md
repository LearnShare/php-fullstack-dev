# 内容完整性要求（核心规则）

## 受众定位

- **所有文档面向零基础学员撰写**
- 语言需循序渐进但内容必须详实完整
- 避免过度简化，确保零基础学员能够完全理解
- 每个概念都要有充分的解释和多个示例

## 内容详细度

**目标**：充分详实、零基础可学

**行数参考**：300-500 行作为详实度范例，视主题复杂度可增减

**必须包含的要素**：
1. 详细概述
2. 特性说明
3. 多种创建/使用方式
4. 完整的语法说明
5. 参数列表（见方法/函数说明规范）
6. 返回值说明（见方法/函数说明规范）
7. 多个完整示例
8. 输出结果说明
9. 注意事项
10. 常见问题
11. 最佳实践
12. 练习任务（3-5 个）

## 方法/函数/API 说明规范

**介绍任何内置方法、函数、模块时，必须包含**：

### 必需格式（三个部分，缺一不可）

1. **语法**（必需）：
   ```markdown
   **语法**：`functionName(type $param1, type $param2 = default): returnType`
   ```
   - 使用反引号包裹完整函数签名
   - 包含所有参数及其类型
   - 包含返回值类型
   - 参数默认值必须明确标注

2. **参数**（必需）：
   ```markdown
   **参数**：
   - `$param1`：参数说明
   - `$param2`：可选，参数说明（如果是可选参数，必须标注"可选"）
   ```
   - 每个参数必须单独列出
   - 参数名使用反引号包裹
   - 必须说明参数的作用和含义
   - 可选参数必须标注"可选"或"默认值"

3. **返回值**（必需）：
   ```markdown
   **返回值**：返回值的详细说明。
   ```
   - 必须说明返回值的类型
   - 必须说明返回值的含义
   - 特殊情况必须说明（如返回 `false` 表示失败）
   - 如果可能返回多种类型，必须说明所有情况

### 完整示例格式

```markdown
### functionName() - 函数说明

**语法**：`functionName(type $param1, type $param2 = default): returnType`

**参数**：
- `$param1`：参数说明
- `$param2`：可选，参数说明

**返回值**：返回值的详细说明。

```php
// 示例代码
```
```

### 其他要素（按需包含）

4. **完整示例代码**（推荐）：
   ```php
   $result = functionName('value', 'option');
   var_dump($result); // 输出说明
   ```

5. **输出结果说明**（推荐）：
   - 明确说明代码执行后的输出
   - 解释输出含义

6. **注意事项**（推荐）：
   - 使用限制
   - 性能考虑
   - 兼容性说明

7. **常见错误**（推荐）：
   - 常见陷阱
   - 错误示例及修正方法

**重要提示**：所有函数文档必须包含**语法、参数、返回值**三个部分，这是强制要求，缺一不可。详见 [函数文档格式标准](13-function-documentation-standards.md)。

## 对比说明

- 提供与其他数据结构、方法的对比
- 说明使用场景和选择建议
- 示例：`array_map()` vs `array_walk()` vs `foreach`

## 练习任务

**每个章节必须包含 3-5 个练习任务**：
- 练习应覆盖章节核心知识点
- 从简单到复杂，循序渐进
- 提供明确的练习目标
- 示例：
  ```markdown
  ## 练习

  1. 使用 `array_map()` 方法将数组 `[1, 2, 3]` 中的每个元素乘以 2。
  2. 使用 `array_filter()` 方法筛选出数组中的偶数。
  3. 使用 `array_reduce()` 方法计算数组元素的总和。
  ```
