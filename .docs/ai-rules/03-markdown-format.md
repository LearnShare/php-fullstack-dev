# Markdown 格式规范

## 标题与空行

**必须遵守**：
- 标题（`#`、`##`、`###` 等）前后必须各留一空行
- 代码块前后必须各留一空行
- 段落之间必须留一空行
- 列表与上一段之间必须留空行
- 列表项内如含多段文本，段间亦需空行

**示例**：
```markdown
## 标题

这是段落内容。

```php
$code = 'example';
```

这是另一个段落。
```

## 表格对齐（重要）

**核心要求**：**表格必须在 Markdown 源文件中按 `|` 进行列对齐，以方便直接阅读原始文本。**

**必须遵守**：
- 表格列宽必须在源文件中对齐，使用空格填充，确保所有行的 `|` 在同一列位置对齐
- 必须使用对齐标记：
  - 文本列左对齐：`|:---|`
  - 数值列右对齐：`|---:|`
  - 可选居中：`|:---:|`
- 宽度比例：**9 个中文字符 ≈ 15 个英文字符**（用于计算列宽）
- 分隔线必须与列宽对齐
- 行尾无多余空格

**目的**：让 Markdown 源文件本身具有良好的可读性，无需渲染即可清晰查看表格结构。

**错误示例**：
```markdown
| 特性 | Laravel | Symfony |
|------|---------|---------|
| 学习 | 简单   | 复杂   |
```

**正确示例**：
```markdown
| 特性                 | Laravel | Symfony |
|:---------------------|:--------|:--------|
| **学习曲线**         | 简单    | 复杂    |
| **性能**             | 良好    | 优秀    |
```

## 代码块格式

**必须遵守**：
- 使用标准 Markdown 代码块格式：```` ```php ````
- **禁止使用** CODE REFERENCES 格式（如 ````startLine:endLine:filepath）
- 代码块前后各留一空行
- **语言标识必须使用简写**：`php`、`bash`（不是 `shell`）、`json`、`md`、`sql` 等
- 代码内容保持原样，不额外插入无关注释或空格

**代码块标注**（推荐）：
- 在代码块内使用注释标注文件路径、行号、上下文（如适用）
- 示例：
```php
// 文件: src/Utils/Helper.php
// 行号: 10-15
// 功能: 格式化日期

function formatDate(DateTime $date): string
{
    return $date->format('Y-m-d');
}
```

**行内代码**：
- 使用反引号包裹：`` `inline` ``
- 避免与周围标点粘连

## 行宽控制

**必须遵守**：
- 避免过长行（建议 <100 字符）
- 必要时采用列表、子标题等替代表达
- 段落不宜过长（建议 <6 行）
- 超长信息拆成多行或子项

## 列表格式

**必须遵守**：
- 无序列表使用 `- `（短横线 + 空格）
- 有序列表使用 `1. `（数字 + 点 + 空格）
- 列表条目简洁；多级内容用子列表
- 列表与上一段之间留空行
