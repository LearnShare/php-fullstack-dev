# 执行工作流

## 执行前

**必须完成**：
- 阅读本规则文档体系（`.docs/ai-rules/` 目录下的所有规则文件）
- 阅读相关章节的现有内容
- 理解任务要求和上下文
- 确认项目结构和文件命名规范

**检查清单**：
- [ ] 已阅读相关规则文件
- [ ] 已了解任务上下文
- [ ] 已确认文件命名和目录结构

## 执行中

**必须遵守**：
- 严格遵守所有规则（格式规范、内容规范、代码规范等）
- 不擅自删改用户已有内容
- 如需要修改，先说明计划并展示变更范围
- 使用专门的编辑工具进行文件操作，不直接在终端命令中修改文件

**工作原则**：
- 逐个处理，禁止使用 grep/脚本进行批量操作（除非用户明确要求）
- 建立 TODO 持续自我跟踪处理进度
- 复杂任务需分阶段完成，每完成一个阶段进行验证

## 执行后

**必须完成**：
- 自检与总结，确保符合规范
- 对照 [自检清单](11-checklist.md) 逐项确认
- 复杂任务需自我验收，尽量减少用户额外干预
- 写入文件后，必须再次读取验证内容是否正确

**验证步骤**：
- [ ] 文件已成功写入
- [ ] 内容符合所有规范要求
- [ ] 引用路径正确
- [ ] 格式规范已遵守（特别是表格对齐、标题空行等）
