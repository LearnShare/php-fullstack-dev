# 工具与操作规范

## 概述

本文档定义了文件操作、命令执行、输出方式等工具使用规范。

---

## 文件修改

**必须遵守**：

- **不直接在 `run_terminal_cmd` 中修改文件**
- 若需修改，先说明计划并展示变更范围
- 使用专门的编辑工具进行文件操作

---

## 输出方式

**规则**：

- 避免在终端使用 `echo` 输出说明
- 优先使用 Markdown 格式输出
- 对话和文档内容要稳重、简洁、精确，尽量避免 Emoji

---

## 执行原则

**工作原则**：

- 逐个处理，禁止使用 grep/脚本进行批量操作（除非用户明确要求）
- 建立 TODO 持续自我跟踪处理进度
- 复杂任务需分阶段完成，每完成一个阶段进行验证
