# 内容完整性要求

## 概述

本文档定义了所有章节文档的内容完整性要求。**所有文档必须面向零基础学员，内容详实完整，确保读者能够完全理解。**

## 核心编写原则

### 客观性

- 文档必须保持客观、严谨、简洁
- 严禁使用任何情感化修饰语（如：伟大的、惊人的、极其、极致）

### 去隐喻化

- 禁止使用非技术性的比喻（如：灵魂、心脏、英雄、基石、天花板）

### 去黑话化

- 禁止使用营销类或非技术背景的词汇（如：赋能、闭环、抓手、工业级实战、秘密武器、阿里味词汇）

### 零起点逻辑

- 文档应适合从零开始学习
- 所有技术概念必须提供详尽的逻辑推导和原理分析
- 确保读者能建立完整的知识框架

### 专业技术标准

- 文档必须符合专业技术文档标准
- 内容详实且完整

## 受众定位

- **所有文档面向零基础学员撰写**
- 语言需循序渐进但内容必须详实完整
- 避免过度简化，确保零基础学员能够完全理解
- 每个概念都要有充分的解释和多个示例

## 内容详细度

**目标**：充分详实、零基础可学

**行数参考**：300-500 行作为详实度范例，视主题复杂度可增减

## 内容完整性要求（12 要素）

每个章节文档必须完整覆盖以下 12 个要素：

| 序号 | 要素名称         | 逻辑说明                                         |
|:-----|:----------------|:-------------------------------------------------|
| 1    | **概述**         | 准确定义技术点，说明其在系统中的定位。           |
| 2    | **特性**         | 列表说明核心技术优势与功能特点。                 |
| 3    | **语法 / 架构**  | 提供物理层面的架构图或核心语法声明。             |
| 4    | **参数详述**     | 穷尽式解析 API 参数、配置项及环境变量（表格化）。|
| 5    | **返回值 / 状态** | 明确标注函数返回值类型及系统反馈的状态码信息。   |
| 6    | **代码示例**     | 包含完整、可运行的代码片段（带文件路径标注）。   |
| 7    | **输出结果说明** | 记录代码运行后的真实控制台输出及其逻辑解析。     |
| 8    | **注意事项**     | 描述边缘情况、异常处理逻辑及跨平台差异。         |
| 9    | **常见问题 (FAQ)**| 针对学习中的疑惑提供针对性的逻辑解答。           |
| 10   | **最佳实践**     | 提供工业级的避坑指南和架构建议。                 |
| 11   | **对比分析**     | 与相似技术或其他实现方式的性能对比。             |
| 12   | **练习任务**     | 设计 3-5 个具有逻辑深度的调研或实践任务。        |

## 方法/函数/API 说明规范

**介绍任何内置方法、函数、模块时，必须包含**：

### 必需格式（三个部分，缺一不可）

1. **语法**（必需）：
   ```markdown
   **语法**：`functionName(type $param1, type $param2 = default): returnType`
   ```
   - 使用反引号包裹完整函数签名
   - 包含所有参数及其类型
   - 包含返回值类型
   - 参数默认值必须明确标注

2. **参数**（必需）：
   ```markdown
   **参数**：
   - `$param1`：参数说明
   - `$param2`：可选，参数说明（如果是可选参数，必须标注"可选"）
   ```
   - 每个参数必须单独列出
   - 参数名使用反引号包裹
   - 必须说明参数的作用和含义
   - 可选参数必须标注"可选"或"默认值"

3. **返回值**（必需）：
   ```markdown
   **返回值**：返回值的详细说明。
   ```
   - 必须说明返回值的类型
   - 必须说明返回值的含义
   - 特殊情况必须说明（如返回 `false` 表示失败）
   - 如果可能返回多种类型，必须说明所有情况

### 完整示例格式

```markdown
### functionName() - 函数说明

**语法**：`functionName(type $param1, type $param2 = default): returnType`

**参数**：
- `$param1`：参数说明
- `$param2`：可选，参数说明

**返回值**：返回值的详细说明。

```php
// 示例代码
```
```

### 其他要素（按需包含）

4. **完整示例代码**（推荐）：
   ```php
   $result = functionName('value', 'option');
   var_dump($result); // 输出说明
   ```

5. **输出结果说明**（推荐）：
   - 明确说明代码执行后的输出
   - 解释输出含义

6. **注意事项**（推荐）：
   - 使用限制
   - 性能考虑
   - 兼容性说明

7. **常见错误**（推荐）：
   - 常见陷阱
   - 错误示例及修正方法

**重要提示**：所有函数文档必须包含**语法、参数、返回值**三个部分，这是强制要求，缺一不可。详见 [函数文档格式标准](07-function-documentation.md)。

## 对比说明

- 提供与其他数据结构、方法的对比
- 说明使用场景和选择建议
- 示例：`array_map()` vs `array_walk()` vs `foreach`

## 练习任务

**每个章节必须包含 3-5 个练习任务**：
- 练习应覆盖章节核心知识点
- 从简单到复杂，循序渐进
- 提供明确的练习目标

**示例**：

```markdown
## 练习

1. 使用 `array_map()` 方法将数组 `[1, 2, 3]` 中的每个元素乘以 2。
2. 使用 `array_filter()` 方法筛选出数组中的偶数。
3. 使用 `array_reduce()` 方法计算数组元素的总和。
```

## 技术栈规范

### PHP 版本

- 所有代码示例必须使用 PHP 8.2+ 语法特性

### 类型声明

- 强制使用类型声明（参数类型、返回类型、属性类型）

### PSR 标准

- 代码必须符合 PSR-12 编码规范
- 命名空间使用 PSR-4 自动加载标准

### 现代特性

- 优先使用 PHP 8+ 新特性（如：构造器属性提升、枚举、只读属性等）

## 关联与引用

### 避免冗余

- 当涉及 Web 前端基础知识时，应通过链接进行关联，不进行重复解释

### 交叉引用

- 合理使用章节间的交叉引用，帮助读者建立知识体系
