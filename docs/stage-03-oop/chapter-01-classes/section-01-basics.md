# 3.1.1 类与对象基础

## 概述

类与对象是面向对象编程的基础。本节介绍类定义、对象创建、属性定义与访问、方法定义、`$this` 关键字等基础知识，帮助理解面向对象编程的核心概念。

**章节类型**：语法性章节

**主要内容**：
- 类的定义语法
- 对象的创建（实例化）
- 属性的定义和访问
- 方法的定义和调用
- `$this` 关键字的使用
- 对象与类的区别
- 基础示例和练习

## 特性

- **类**：对象的模板，定义属性和方法
- **对象**：类的实例，拥有独立的属性值
- **封装**：通过类封装数据和行为

## 语法/定义

### 类定义

**语法**：`class ClassName { ... }`
**特点**：类名使用 `StudlyCase` 命名

### 对象创建

**语法**：`$object = new ClassName();`
**特点**：使用 `new` 关键字创建对象

### 属性访问

**语法**：`$object->property`
**特点**：使用 `->` 访问对象属性

### 方法调用

**语法**：`$object->method()`
**特点**：使用 `->` 调用对象方法

## 基本用法

### 类定义示例

```php
<?php
declare(strict_types=1);

class User
{
    public string $name;
    public int $age;
    
    public function greet(): string
    {
        return "Hello, {$this->name}!";
    }
}
```

### 对象创建示例

```php
<?php
declare(strict_types=1);

$user = new User();
$user->name = "John";
$user->age = 25;
echo $user->greet();
```

## 使用场景

- **数据封装**：将相关数据和行为封装在一起
- **代码复用**：通过类复用代码
- **模块化**：将功能模块化

## 注意事项

- **类名规范**：使用 `StudlyCase` 命名
- **属性初始化**：属性需要初始化或使用默认值
- **$this 使用**：只能在方法内部使用 `$this`

## 常见问题

### 问题 1：$this 未定义
- **原因**：在静态方法或类外部使用 `$this`
- **解决**：只在实例方法中使用 `$this`

### 问题 2：属性未初始化
- **原因**：访问未初始化的属性
- **解决**：在构造器中初始化或使用默认值

## 最佳实践

- 使用有意义的类名
- 合理设计类的属性和方法
- 理解类与对象的关系
- 遵循 PSR 命名规范
