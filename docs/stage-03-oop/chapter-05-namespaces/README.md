# 3.5 命名空间与自动加载

## 目标

- 理解命名空间（Namespace）的作用与语法，掌握如何避免类名冲突。
- 熟悉 `use` 语句的使用方式，包括别名、分组导入等。
- 掌握 PSR-4 自动加载标准，理解 Composer 自动加载的工作原理。
- 能够组织大型项目的目录结构，实现模块化的代码组织。

## 章节内容

本章分为四个独立小节，每节提供详细的概念解释、语法说明、参数列表和完整示例：

1. **[命名空间基础](section-01-basics.md)**：为什么需要命名空间、命名空间语法、子命名空间、全局命名空间、命名空间解析规则及完整示例。

2. **[use 语句](section-02-use-statements.md)**：基础导入、别名（Alias）、分组导入、函数和常量导入及完整示例。

3. **[自动加载基础](section-03-autoloading-basics.md)**：传统方式的问题、`spl_autoload_register`、简单自动加载器示例、多个自动加载器及完整示例。

4. **[PSR-4 自动加载标准](section-04-psr4.md)**：PSR-4 规则、Composer 自动加载配置、目录结构示例、自动加载优化、调试自动加载、命名空间最佳实践及完整示例。

## 核心概念

- **命名空间**：解决类名冲突，组织代码结构
- **use 语句**：导入类、函数、常量
- **自动加载**：按需加载类文件
- **PSR-4**：标准的自动加载规范

## 学习建议

1. **按顺序学习**：按顺序学习四个小节，理解命名空间和自动加载的完整体系。

2. **重点掌握**：
   - 命名空间的语法和使用
   - use 语句的各种用法
   - 自动加载的工作原理
   - PSR-4 标准的配置

3. **实践练习**：
   - 完成每小节后的练习题目
   - 创建项目结构并配置自动加载
   - 理解命名空间与目录的关系

## 完成本章后

- 能够使用命名空间组织代码，避免类名冲突。
- 能够使用 use 语句导入类、函数和常量。
- 能够理解自动加载的工作原理。
- 能够配置和使用 PSR-4 自动加载标准。
