# 3.8 六边形架构

## 目标

- 理解六边形架构（Hexagonal Architecture / Ports & Adapters）的核心概念。
- 掌握 Port（端口）和 Adapter（适配器）的设计原则。
- 理解如何通过六边形架构实现业务逻辑与技术实现的解耦。
- 能够在实际项目中应用六边形架构，设计可测试、可扩展的应用。

## 章节内容

本章分为三个独立小节，每节提供详细的概念解释、语法说明、参数列表和完整示例：

1. **[六边形架构概述与 Ports](section-01-overview-ports.md)**：核心思想、为什么需要六边形架构、输入端口、输出端口、核心实现及完整示例。

2. **[Adapters（适配器）](section-02-adapters.md)**：输入适配器（HTTP、CLI）、输出适配器（数据库、邮件服务、测试适配器）、依赖注入配置及完整示例。

3. **[六边形架构实践](section-03-practice.md)**：完整项目结构、依赖注入配置、测试策略、最佳实践及完整示例。

## 核心概念

- **核心（Core）**：业务逻辑，不依赖外部技术
- **端口（Ports）**：接口定义，描述应用需求
- **适配器（Adapters）**：技术实现，连接外部世界

## 学习建议

1. **按顺序学习**：按顺序学习三个小节，理解六边形架构的完整体系。

2. **重点掌握**：
   - 端口的设计原则
   - 适配器的实现方式
   - 依赖注入的配置

3. **实践练习**：
   - 完成每小节后的练习题目
   - 实现完整的六边形架构应用
   - 创建测试适配器

## 完成本章后

- 能够理解六边形架构的核心概念和设计原则。
- 能够设计端口和适配器，实现业务逻辑与技术实现的解耦。
- 能够应用六边形架构设计可测试、可扩展的应用。
- 能够使用依赖注入管理端口和适配器。
