# 3.7 应用架构模式

## 目标

- 理解 MVC（Model-View-Controller）架构模式的核心概念与实现方式。
- 掌握 ADR（Action-Domain-Responder）模式，理解其与 MVC 的区别。
- 熟悉模块化单体架构的组织方式（Domain/Infrastructure/Application 分层）。
- 能够在实际项目中应用这些架构模式，设计清晰的应用结构。

## 章节内容

本章分为三个独立小节，每节提供详细的概念解释、语法说明、参数列表和完整示例：

1. **[MVC 架构模式](section-01-mvc.md)**：MVC 核心概念、基础实现（Model、View、Controller）、路由示例、优缺点分析及完整示例。

2. **[ADR 架构模式](section-02-adr.md)**：ADR 核心概念、实现示例（Domain、Action、Responder）、ADR vs MVC 对比、优势分析及完整示例。

3. **[模块化单体架构](section-03-modular-monolith.md)**：分层架构、目录结构组织、依赖方向、实际应用示例、模块化单体优势及完整示例。

## 核心概念

- **MVC**：Model-View-Controller，经典的三层架构
- **ADR**：Action-Domain-Responder，MVC 的改进版本
- **模块化单体**：分层架构，模块边界清晰

## 学习建议

1. **按顺序学习**：按顺序学习三个小节，理解不同架构模式的特点。

2. **重点掌握**：
   - MVC 的基本实现
   - ADR 与 MVC 的区别
   - 模块化单体的分层结构

3. **实践练习**：
   - 完成每小节后的练习题目
   - 实现完整的 MVC 和 ADR 应用
   - 设计模块化单体架构

## 完成本章后

- 能够实现基本的 MVC 架构应用。
- 能够理解和使用 ADR 架构模式。
- 能够设计模块化单体架构，组织清晰的代码结构。
- 能够根据项目需求选择合适的架构模式。
