# 3.6 异常体系与框架级设计

## 目标

- 深入理解 PHP 异常体系（`Throwable`、`Exception`、`Error`）的层次结构。
- 掌握自定义异常的创建与使用，设计语义化的异常类层次。
- 理解框架级异常设计原则，能够构建可维护的异常处理体系。
- 熟悉异常传播、捕获策略与日志记录的最佳实践。

## 章节内容

本章分为三个独立小节，每节提供详细的概念解释、语法说明、参数列表和完整示例：

1. **[异常体系层次](section-01-exception-hierarchy.md)**：Throwable 接口、Exception vs Error、异常类层次、异常捕获及完整示例。

2. **[自定义异常](section-02-custom-exceptions.md)**：基础自定义异常、异常属性扩展、异常链、异常分类策略及完整示例。

3. **[框架级异常设计](section-03-framework-design.md)**：异常处理策略、异常转换、异常处理中间件、框架级异常设计、异常与日志、最佳实践及完整示例。

## 核心概念

- **Throwable**：所有可抛出对象的根接口
- **Exception**：业务逻辑错误
- **Error**：系统级错误
- **异常链**：保留完整的错误上下文

## 学习建议

1. **按顺序学习**：按顺序学习三个小节，理解异常体系的完整设计。

2. **重点掌握**：
   - 异常体系层次结构
   - 自定义异常的设计
   - 框架级异常处理策略

3. **实践练习**：
   - 完成每小节后的练习题目
   - 设计异常类层次结构
   - 实现异常处理中间件

## 完成本章后

- 能够理解 PHP 异常体系的层次结构。
- 能够创建语义化的自定义异常类。
- 能够设计框架级的异常处理体系。
- 能够应用异常处理最佳实践。
