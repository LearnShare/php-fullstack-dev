# 3.3 OOP 三大特性

## 目标

- 理解继承（Inheritance）的概念，掌握 `extends` 关键字与 `final` 关键字的使用。
- 掌握接口（Interface）的定义与实现，理解接口作为契约的作用。
- 熟悉抽象类（Abstract Class）的使用场景，理解抽象类与接口的区别。
- 理解多态（Polymorphism）的概念，能够在实际项目中应用多态设计。

## 章节内容

本章分为四个独立小节，每节提供详细的概念解释、语法说明、参数列表和完整示例：

1. **[3.3.1 继承（Inheritance）](section-01-inheritance.md)**：基础语法、`extends` 关键字、访问父类方法、`parent` 关键字、构造函数继承、`final` 关键字、继承层次示例及完整示例。

2. **[3.3.2 接口（Interface）](section-02-interfaces.md)**：基础语法、接口方法可见性、实现多个接口、接口继承、接口作为类型提示、多态实现及完整示例。

3. **[3.3.3 抽象类（Abstract Class）](section-03-abstract-classes.md)**：基础语法、抽象方法、抽象类 vs 接口、模板方法模式、使用场景及完整示例。

4. **[3.3.4 多态（Polymorphism）](section-04-polymorphism.md)**：多态概念、在依赖注入中的应用、完整示例（支付系统）、多态优势、注意事项及完整示例。

## 核心概念

- **继承**：代码复用和层次化设计
- **接口**：定义契约，实现多态
- **抽象类**：提供部分实现的模板
- **多态**：同一接口，不同实现

## 学习建议

1. **按顺序学习**：按顺序学习四个小节，理解 OOP 三大特性的完整体系。

2. **重点掌握**：
   - 继承的使用和限制
   - 接口的定义和实现
   - 抽象类与接口的区别
   - 多态的实现和应用

3. **实践练习**：
   - 完成每小节后的练习题目
   - 设计类层次结构
   - 应用多态解决实际问题

## 完成本章后

- 能够使用继承实现代码复用和层次化设计。
- 能够定义和实现接口，理解接口作为契约的作用。
- 能够使用抽象类提供部分实现的模板。
- 能够应用多态提高代码的灵活性和可扩展性。
