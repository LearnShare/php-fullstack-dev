# 6.10 可观测性体系

## 目标

- 理解可观测性的三大支柱：日志、指标、链路追踪。
- 了解 OpenTelemetry 的使用。
- 熟悉 Prometheus 指标暴露和 Grafana 可视化。
- 掌握 Sentry 错误监控的配置。

## 章节内容

本章分为三个独立小节，每节提供详细的概念解释、代码示例和最佳实践：

1. **[可观测性概述](section-01-overview.md)**：可观测性概念、三大支柱、可观测性 vs 监控、完整示例。

2. **[日志、指标、追踪](section-02-logs-metrics-traces.md)**：结构化日志、Prometheus 指标、OpenTelemetry 追踪、完整示例。

3. **[监控与告警](section-03-monitoring-alerts.md)**：监控系统、告警配置、可视化、完整示例。

## 核心概念

- **可观测性**：系统可观测程度
- **三大支柱**：日志、指标、追踪
- **OpenTelemetry**：可观测性标准
- **监控告警**：系统监控和告警

## 学习建议

1. **重点掌握**：
   - 三大支柱的使用
   - OpenTelemetry 配置
   - 监控告警

2. **实践练习**：
   - 完成每小节后的练习题目
   - 建立可观测性体系
   - 配置监控告警

## 完成本章后

- 能够建立完整的可观测性体系。
- 理解三大支柱，能够收集和分析数据。
- 掌握监控告警，能够及时发现问题。
- 具备构建生产级可观测性系统的能力。
