# 2.2 输出与调试基础

## 目标

- 掌握 `echo`、`print`、`printf`、`sprintf` 的语义差异，新手可以在 CLI 和浏览器中正确输出信息。
- 理解 `var_dump`、`print_r`、`debug_zval_dump` 等调试函数，能读懂输出结构。
- 构建"开发输出 vs 生产日志"的思维，避免把调试信息泄漏到用户界面。

## 章节内容

本章分为三个独立小节，每节提供详细的概念解释、语法说明、参数列表和完整示例：

1. **[2.2.1 输出 API](section-01-output-api.md)**：`echo`、`print`、`printf`、`sprintf` 及输出缓冲机制详解。

2. **[2.2.2 格式化占位符详解](section-02-format-placeholders.md)**：`printf` 和 `sprintf` 的格式化占位符语法、类型、参数位置指定及实际应用场景。

3. **[2.2.3 调试函数和策略](section-03-debugging.md)**：`var_dump`、`print_r`、`debug_backtrace` 等调试函数，以及开发/生产环境的调试策略（仅 CLI 环境）。

## 核心概念

- **输出函数**：`echo`、`print`、`printf`、`sprintf` 的区别和用法
- **格式化占位符**：`%s`、`%d`、`%f` 等占位符的使用
- **调试函数**：`var_dump`、`print_r`、`debug_backtrace` 的使用场景
- **输出缓冲**：输出缓冲机制的作用和使用

## 学习建议

1. **按顺序学习**：按顺序学习三个小节，每完成一节即编写 2-3 个小练习。

2. **重点掌握**：
   - `sprintf` 的格式化占位符，这是实际开发中最常用的功能
   - 输出缓冲机制，为后续模板渲染、邮件生成等场景打下基础
   - 开发/生产环境的调试策略，避免在生产环境泄漏调试信息

3. **实践练习**：
   - 练习使用不同的输出函数
   - 练习使用格式化占位符
   - 练习使用调试函数

## 完成本章后

- 能够根据场景选择合适的输出函数（`echo` vs `printf` vs `sprintf`）。
- 熟练使用格式化占位符生成订单号、日志格式、表格输出等。
- 掌握输出缓冲机制，能够捕获模板输出、生成邮件内容。
- 理解调试函数的使用场景，能够使用调试函数进行问题排查。
- 具备开发/生产环境的调试策略意识，能够安全地处理调试信息。
