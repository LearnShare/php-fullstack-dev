# 2.14 PHP 版本新特性

## 目标

- 全面掌握 PHP 8.2 ~ 8.5 的语法特性、运行时改进与工具生态更新。
- 了解新特性的适用场景、注意事项与迁移成本，提升代码现代化程度。
- 构建自上而下的升级策略，确保生产环境平滑过渡。

## 章节内容

本章分为五个独立小节，每节提供详细的概念解释、语法说明、参数列表和完整示例：

1. **[2.14.1 PHP 8.2 新特性与示例](section-01-php82.md)**：Readonly 类、独立类型、枚举常量表达式、敏感参数、全新 Random 扩展等重点能力。

2. **[2.14.2 PHP 8.3 新特性与示例](section-02-php83.md)**：`json_validate()`、类常量类型、`#[\Override]`、`mb_str_pad()`、覆盖属性类型等语法增强。

3. **[2.14.3 PHP 8.4 新特性与示例](section-03-php84.md)**：属性钩子、不对称可见性、`array_first/array_last`、`#[\NoDiscard]`、DOM API 改进等。

4. **[2.14.4 PHP 8.5 新特性与示例](section-04-php85.md)**：管道操作符、`clone with`、URI parser 配置支持、OPcache 必选、废弃非规范化类型转换等。

5. **[2.14.5 升级指南与最佳实践](section-05-upgrade-best-practices.md)**：逐版本升级路线、兼容性检测脚本、最佳实践与练习任务。

## 学习路径

- 建议先阅读当前项目所处版本的章节，再向上了解未来版本差异。
- 每学习一个特性，都动手编写"旧写法 vs 新写法"的对照练习，巩固迁移思路。
- 升级前根据第五节的清单完成预检查，上线后持续观察指标与日志。

## 学习建议

1. **按顺序学习**：按顺序学习五个小节，了解各版本的新特性。

2. **重点掌握**：
   - PHP 8.2+ 的新语法特性
   - 新特性的适用场景和注意事项
   - 升级策略和兼容性检测
   - 迁移最佳实践

3. **实践练习**：
   - 完成每小节后的练习题目
   - 在现有项目中使用新特性
   - 制定升级计划

## 完成本章后

- 了解 PHP 8.2-8.5 的新特性。
- 能够在项目中使用新特性提升代码质量。
- 能够制定和执行升级计划。
- 能够处理升级过程中的兼容性问题。
