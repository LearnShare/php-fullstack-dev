# 2.1 PHP 基本语法结构

## 目标

- 理解 PHP 文件的起始标签、编码要求与执行方式，新手也能独立创建首个 PHP 程序。
- 熟悉语句、分号、缩进、注释的最佳实践，养成良好代码风格。
- 为后续模块化、自动加载奠定一致的代码结构。

## 章节内容

本章分为六个独立小节，每节提供详细的概念解释、语法说明、参数列表和完整示例：

1. **[2.1.1 第一个 PHP 程序：Hello World](section-01-hello-world.md)**：创建并执行第一个 PHP 程序，了解 CLI 和 Web 两种执行方式。

2. **[2.1.2 文件结构与编码](section-02-file-structure.md)**：PHP 起始标签、UTF-8 编码、严格类型声明、命名空间基础。

4. **[2.1.4 文件执行方式](section-04-execution.md)**：CLI 模式、Web 模式、执行环境差异。

## 核心概念

- **PHP 起始标签**：`<?php` 是 PHP 代码的开始标记
- **严格类型声明**：`declare(strict_types=1);` 启用严格类型检查
- **UTF-8 编码**：PHP 文件应使用 UTF-8 编码
- **语句规则**：PHP 语句以分号结尾
- **注释类型**：单行注释、多行注释、PHPDoc 注释

## 学习建议

1. **按顺序学习**：按顺序学习六个小节，每完成一节即编写 2-3 个小练习。

2. **重点掌握**：
   - 文件结构规范（严格类型、编码、命名空间）
   - 语句规则和注释用法
   - 常见错误的识别和解决

3. **实践练习**：
   - 创建并执行第一个 PHP 程序
   - 练习使用不同的注释类型
   - 练习识别和修复语法错误

## 完成本章后

- 能够创建并执行一个规范的 PHP 文件。
- 理解 `<?php` 标签、UTF-8 编码、严格类型声明的作用。
- 掌握语句分号的使用规则和三种注释类型。
- 了解 PSR-12 代码风格要求，能够使用 EditorConfig 配置。
- 能够使用 `php -l` 检查语法错误。
- 理解 CLI 和 Web 执行环境的差异。
- 能够识别并解决常见的语法错误。
