# 2.1.3 语句与注释

## 概述

语句和注释是 PHP 代码的基本组成部分。本节介绍 PHP 语句的规则、分号的使用、三种注释类型（单行、多行、PHPDoc）、PSR-12 代码风格要求，帮助养成规范的代码风格。

**章节类型**：语法性章节

**主要内容**：
- 语句规则和分号使用
- 单行注释（`//`）
- 多行注释（`/* */`）
- PHPDoc 注释
- PSR-12 代码风格要求
- 注释最佳实践

## 特性

- **语句规则**：PHP 语句以分号结尾
- **注释类型**：单行、多行、PHPDoc 三种注释
- **代码风格**：遵循 PSR-12 标准

## 语法/定义

### 语句规则

**基本规则**：
- PHP 语句以分号（`;`）结尾
- 最后一个语句的分号可以省略（不推荐）
- 控制结构不需要分号

### 注释类型

**单行注释**：
- 语法：`// 注释内容`
- 用途：简短说明、临时禁用代码

**多行注释**：
- 语法：`/* 注释内容 */`
- 用途：多行说明、代码块注释

**PHPDoc 注释**：
- 语法：`/** 注释内容 */`
- 用途：函数、类、属性的文档注释

## 基本用法

### 语句示例

```php
<?php
declare(strict_types=1);

$name = "World";  // 语句以分号结尾
echo "Hello, {$name}!\n";  // 输出语句
```

### 注释示例

```php
<?php
declare(strict_types=1);

// 单行注释：这是变量定义
$name = "World";

/* 
 * 多行注释：
 * 这是输出语句
 */
echo "Hello, {$name}!\n";

/**
 * PHPDoc 注释：
 * 这是函数定义
 * 
 * @param string $name 名称
 * @return string 问候语
 */
function greet(string $name): string
{
    return "Hello, {$name}!\n";
}
```

## 使用场景

- **单行注释**：简短说明、临时禁用代码
- **多行注释**：详细说明、代码块注释
- **PHPDoc**：函数、类、属性的文档

## 注意事项

- **分号规则**：语句必须加分号，控制结构不需要
- **注释规范**：使用有意义的注释，避免无意义的注释
- **代码风格**：遵循 PSR-12 标准
- **PHPDoc**：为公共 API 编写完整的 PHPDoc

## 常见问题

### 问题 1：缺少分号
- **原因**：语句未加分号
- **症状**：语法错误
- **解决**：检查并添加分号

### 问题 2：注释格式不规范
- **原因**：未遵循 PSR-12 标准
- **解决**：使用代码格式化工具

## 最佳实践

- 所有语句都加分号
- 使用有意义的注释
- 为公共 API 编写 PHPDoc
- 遵循 PSR-12 代码风格
- 使用 EditorConfig 统一代码风格
