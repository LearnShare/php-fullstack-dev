# 2.12 文件引入与模块化

## 目标

- 理解如何编写和定义独立的文件/模块。
- 掌握如何导入其他文件/模块，以及如何使用导入的内容。
- 学会避免导入内容的冲突和循环导入问题。
- 理解命名空间在模块化中的应用方式。
- 掌握 Composer 的基本使用，包括安装和导入第三方模块。
- 学会编写和发布 Composer 模块。

## include / require 对比

| 关键字      | 失败时行为         | 返回值 |
| :---------- | :----------------- | :----- |
| `include`   | 发出警告 (`E_WARNING`)，脚本继续执行 | 成功返回 1，失败返回 `false` |
| `require`   | 抛出致命错误 (`E_ERROR`)，脚本终止 | 成功返回 1，失败不返回（脚本终止） |
| `include_once` | 仅在第一次包含文件时执行，避免重复定义 | 同 `include` |
| `require_once` | 同上，但致命错误行为不变 | 同 `require` |

> **建议**：对关键依赖使用 `require_once`，模板或可选模块用 `include`。

## 学习建议

1. **按顺序学习**：
   - 先理解模块基础和文件引入（1-2 节）
   - 再学习如何使用导入的内容和避免问题（3-5 节）
   - 然后学习 Composer 的使用（6-8 节）
   - 最后深入学习命名空间（9 节）

2. **重点掌握**：
   - 如何编写独立的模块文件
   - `include` 和 `require` 的区别和使用场景
   - 如何避免导入内容的冲突
   - 如何避免循环导入
   - 命名空间的定义和使用
   - Composer 的安装和使用
   - 如何安装和使用第三方包
   - 如何编写和发布自己的包
   - PSR-4 自动加载的配置和使用

3. **实践练习**：
   - 完成每小节后的练习题目
   - 创建多个模块文件，练习导入和使用
   - 配置一个使用 Composer autoload 的项目
   - 安装并使用第三方包
   - 创建一个简单的 Composer 包并准备发布
