# 6.3 并发控制与 MVCC

## 目标

- 理解数据库并发控制的概念和重要性，掌握并发问题的解决方法。
- 理解 MVCC（多版本并发控制）机制，掌握事务隔离级别。
- 掌握锁机制的使用，能够处理并发访问问题。

## 章节内容

本章分为三个独立小节，每节提供详细的概念解释、语法说明、参数列表和完整示例：

1. **[6.3.1 并发控制](section-01-concurrency-control.md)**：并发控制概念、并发问题（脏读、不可重复读、幻读）、事务隔离级别、完整示例。

2. **[6.3.2 MVCC 机制](section-02-mvcc.md)**：MVCC 概念、版本控制、快照读、当前读、完整示例。

3. **[6.3.3 锁机制](section-03-locking.md)**：锁的概念、锁的类型（共享锁、排他锁）、行锁、表锁、死锁处理、完整示例。

## 核心概念

- **并发控制**：管理同时访问数据库的机制
- **MVCC**：多版本并发控制
- **事务隔离级别**：事务之间的隔离程度
- **锁**：控制资源访问的机制

## 学习建议

1. **按顺序学习**：按顺序学习三个小节，理解并发控制的完整体系。

2. **重点掌握**：
   - 并发问题类型
   - 事务隔离级别
   - MVCC 机制
   - 锁的使用

3. **实践练习**：
   - 完成每小节后的练习题目
   - 理解并发问题的场景
   - 掌握锁的使用方法

## 完成本章后

- 能够理解并发控制的重要性。
- 能够选择合适的隔离级别。
- 能够理解 MVCC 机制。
- 能够使用锁处理并发问题。
