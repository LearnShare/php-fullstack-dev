# 第 6.7 章：数据库性能监控

## 章节概述

仅仅让数据库“运行起来”是远远不够的，确保它能“高效地运行”同样至关重要。**数据库性能监控**是主动发现问题、诊断瓶颈、保障系统稳定性的关键环节。一个看似微小的性能问题，在流量高峰期都可能被放大，导致整个应用响应缓慢甚至崩溃。

本章将介绍开发者应该关注的一些核心数据库性能指标，并重点学习 MySQL 中用于诊断慢查询的强大工具。我们将了解如何识别和分析那些拖慢系统速度的 SQL 语句，并探讨一些常见的 SQL 优化思路。虽然深入的性能调优是 DBA 的专业领域，但作为开发者，掌握基本的监控和诊断能力，能让你从代码层面编写出对数据库更友好的、性能更高的查询。

## 学习目标

完成本章学习后，你将能够：

-   **了解核心性能指标**：知道应该关注哪些数据库的关键性能指标（如 QPS, TPS, 慢查询数等）。
-   **掌握慢查询日志的使用**：能够开启并分析 MySQL 的慢查询日志，快速定位有问题的 SQL 语句。
-   **学会使用 EXPLAIN 命令**：能够使用 `EXPLAIN` 来分析 SQL 查询的执行计划，理解 MySQL 是如何执行你的查询的。
-   **识别常见的低效查询**：能够通过执行计划，识别出全表扫描、未使用索引等典型的性能问题。
-   **了解基本的 SQL 优化思路**：掌握一些常见的 SQL 优化技巧，如添加索引、改写查询等。
