# 第 6.2 章：PDO 入门与高安全模式

## 章节概述

在掌握了数据库设计的基础理论之后，我们现在需要一个强大、安全且灵活的工具来让 PHP 应用程序与数据库进行交互。本章将深入讲解 **PDO (PHP Data Objects)**，这是 PHP 官方推荐的、用于访问数据库的现代标准接口。

与传统的 `mysql_*` 或 `mysqli_*` 函数不同，PDO 提供了一个统一的抽象层，这意味着你可以使用同一套 PDO 函数来操作 MySQL、PostgreSQL、SQLite 等多种不同的数据库，只需在连接时更改相应的数据库驱动即可。这极大地提升了代码的可移植性和可维护性。

本章将从 PDO 的基础概念和连接方式讲起，重点介绍其最核心的特性——**预处理语句 (Prepared Statements)**，这是防止 SQL 注入攻击的根本手段。我们还将学习如何执行 CRUD (Create, Read, Update, Delete) 操作、处理事务以及配置 PDO 的错误处理模式，确保你的数据库操作既高效又安全。

## 学习目标

完成本章学习后，你将能够：

-   **理解 PDO 的核心优势**：明确 PDO 相对于旧数据库扩展的优点。
-   **安全地连接到数据库**：掌握使用 DSN 配置并建立 PDO 连接的方法。
-   **精通预处理语句**：能够熟练使用预处理语句来执行查询，从根本上杜绝 SQL 注入风险。
-   **执行完整的 CRUD 操作**：使用 PDO 进行数据的增、删、改、查。
-   **掌握数据获取的多种方式**：学习如何将查询结果集提取为关联数组、索引数组或对象。
-   **管理数据库事务**：能够使用 PDO 的事务处理功能来确保一系列操作的原子性。
-   **配置 PDO 错误处理模式**：理解并能设置合适的错误模式，以便优雅地处理数据库异常。
