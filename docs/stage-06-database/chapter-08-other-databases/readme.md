# 第 6.8 章：其他数据库

## 章节概述

到目前为止，我们的学习重心一直围绕着关系型数据库（RDBMS），特别是 MySQL。关系型数据库以其强大的事务能力和数据一致性保证，在过去几十年里一直是绝大多数应用的数据基石。

然而，随着互联网应用的飞速发展，数据量、并发量和业务需求的多样性都呈爆炸式增长。在某些特定场景下，传统关系型数据库“一刀切”的严格模式和扩展方式开始显得力不从心。为了解决这些新的挑战，一系列被称为 **NoSQL (Not Only SQL)** 的数据库应运而生。

本章将作为你数据库知识体系的一个扩展，简要介绍 NoSQL 世界中的几种主流数据库类型，让你了解在“关系”之外，还有一片广阔的天地。理解这些不同类型数据库的特点和适用场景，将帮助你在未来的技术选型中做出更明智的决策。

## 学习目标

完成本章学习后，你将能够：

-   **理解 NoSQL 的基本概念**：知道 NoSQL 是什么，以及它与关系型数据库的主要区别。
-   **了解 NoSQL 的 CAP 理论**：理解分布式系统设计中的一致性、可用性和分区容错性的权衡。
-   **了解几种主流的 NoSQL 数据库类型**：对文档数据库 (MongoDB)、键值数据库 (Redis)、列式数据库 (HBase) 和图数据库 (Neo4j) 有一个基本的认识。
-   **为不同场景选择合适的数据库**：能够根据业务需求，初步判断一个场景是更适合使用关系型数据库还是某种类型的 NoSQL 数据库。
