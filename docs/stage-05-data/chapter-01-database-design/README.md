# 5.1 数据库设计基础

## 目标

- 理解数据库设计的基本原则和范式理论。
- 掌握 ER 图（实体关系图）的绘制方法。
- 熟悉一、二、三范式的应用场景。
- 了解反范式的适用场景与权衡。
- 掌握 MySQL 字符集与排序规则的选择。

## 章节内容

本章分为三个独立小节，每节提供详细的概念解释、代码示例和最佳实践：

1. **[数据库设计原则](section-01-design-principles.md)**：为什么需要数据库设计、ER 图（实体关系图）、关系类型、数据库设计最佳实践及完整示例。

2. **[范式与反范式](section-02-normalization.md)**：第一范式（1NF）、第二范式（2NF）、第三范式（3NF）、反范式设计、范式与反范式权衡及完整示例。

3. **[索引设计](section-03-index-design.md)**：索引基础、索引类型、索引设计原则、索引优化、完整示例及最佳实践。

## 核心概念

- **ER 图**：实体关系图，用于数据库设计
- **范式**：数据库规范化理论
- **反范式**：为性能优化而适度冗余
- **索引**：提升查询性能的数据结构

## 学习建议

1. **重点掌握**：
   - ER 图的绘制方法
   - 范式的应用场景
   - 索引设计原则

2. **实践练习**：
   - 完成每小节后的练习题目
   - 设计一个完整的数据库结构
   - 分析现有数据库设计

## 完成本章后

- 能够设计规范的数据库结构。
- 理解范式理论，能够合理应用。
- 掌握索引设计，能够优化查询性能。
- 具备数据库设计文档编写能力。
