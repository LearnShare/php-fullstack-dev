# 5.5 并发控制与 MVCC

## 目标

- 理解数据库并发控制的基本概念。
- 掌握快照读与当前读的区别。
- 了解 MVCC（多版本并发控制）的工作原理。
- 理解 Undo Log 的作用。
- 熟悉幻读（Phantom Read）问题及解决方案。

## 章节内容

本章分为三个独立小节，每节提供详细的概念解释、代码示例和最佳实践：

1. **[并发控制](section-01-concurrency-control.md)**：并发问题（脏读、不可重复读、幻读）、并发控制机制、锁机制、完整示例。

2. **[MVCC 机制](section-02-mvcc.md)**：MVCC 原理、版本链、ReadView、快照读与当前读、Undo Log、完整示例。

3. **[锁机制](section-03-locking.md)**：锁类型、行锁、表锁、间隙锁、Next-Key Lock、死锁处理、完整示例。

## 核心概念

- **并发控制**：控制并发事务的执行
- **MVCC**：多版本并发控制机制
- **锁机制**：保证数据一致性
- **死锁**：多个事务相互等待

## 学习建议

1. **重点掌握**：
   - MVCC 的工作原理
   - 锁机制的使用
   - 死锁的处理

   - 完成每小节后的练习题目
   - 实现并发控制机制
   - 处理死锁场景

- 能够理解并发控制机制。
- 掌握 MVCC 的工作原理。
