# 7.3 应用架构模式

## 目标

- 理解 MVC、ADR 等应用架构模式，掌握架构模式的应用。
- 理解模块化单体架构的概念，能够设计模块化的应用结构。
- 能够根据项目需求选择合适的架构模式。

## 章节内容

本章分为四个独立小节，每节提供详细的概念解释、语法说明、参数列表和完整示例：

1. **[7.3.1 MVC 架构模式](section-01-mvc.md)**：MVC 概念、模型（Model）、视图（View）、控制器（Controller）、MVC 实现、完整示例。

2. **[7.3.2 ADR 架构模式](section-02-adr.md)**：ADR 概念、动作（Action）、域（Domain）、响应器（Responder）、ADR 实现、完整示例。

3. **[7.3.3 模块化单体架构](section-03-modular-monolith.md)**：模块化单体概念、模块划分、模块通信、模块化实现、完整示例。

4. **[7.3.4 架构模式对比与选择](section-04-comparison.md)**：架构模式对比、选择原则、迁移策略、完整示例。

## 核心概念

- **MVC**：Model-View-Controller 架构模式
- **ADR**：Action-Domain-Responder 架构模式
- **模块化单体**：模块化的单体应用架构
- **架构模式**：应用的整体结构模式

## 学习建议

1. **按顺序学习**：按顺序学习四个小节，理解应用架构模式的完整体系。

2. **重点掌握**：
   - MVC 架构
   - ADR 架构
   - 模块化设计
   - 架构选择

3. **实践练习**：
   - 完成每小节后的练习题目
   - 实现不同的架构模式
   - 理解架构模式的应用

## 完成本章后

- 能够理解并应用 MVC 架构。
- 能够理解并应用 ADR 架构。
- 能够设计模块化单体架构。
- 能够根据需求选择合适的架构模式。
