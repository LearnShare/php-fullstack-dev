# 4.10 流量治理与安全

## 目标

- 理解 API Rate Limiting 的作用与实现方式。
- 掌握固定窗口、滑动窗口等限流算法。
- 实现防止重复提交的幂等性机制。
- 了解请求签名（Timestamp + HMAC）的实现。

## 章节内容

本章分为三个独立小节，每节提供详细的概念解释、代码示例和最佳实践：

1. **[Rate Limiting](section-01-rate-limiting.md)**：限流的作用、固定窗口限流、滑动窗口限流、令牌桶算法、漏桶算法、完整示例。

2. **[请求签名与验证](section-02-request-signing.md)**：请求签名原理、HMAC 签名、时间戳验证、签名验证中间件、完整示例。

3. **[安全最佳实践](section-03-security-best-practices.md)**：幂等性机制、防重放攻击、请求验证、安全配置、完整示例。

## 核心概念

- **Rate Limiting**：限制请求频率
- **请求签名**：验证请求完整性
- **幂等性**：防止重复操作
- **防重放**：防止请求重放攻击

## 学习建议

1. **重点掌握**：
   - 限流算法的实现
   - 请求签名的使用
   - 幂等性机制

2. **实践练习**：
   - 完成每小节后的练习题目
   - 实现限流中间件
   - 实现请求签名验证

## 完成本章后

- 能够实现各种限流算法。
- 理解请求签名机制，能够实现签名验证。
