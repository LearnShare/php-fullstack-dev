# 阶段十二：附言（Appendix）

本阶段提供补充参考资料，包括 PSR 标准规范、Composer 高级应用等内容。这些内容作为前面阶段的补充学习资料，帮助提升代码质量和团队协作能力。

## 定位

补充参考资料

## 前置知识要求

在开始本阶段学习之前，请确保你已经：

### 必须掌握（阶段一、阶段二、阶段三）

- **阶段一**：环境搭建、工具配置、Composer 基础
- **阶段二**：PHP 语言基础、代码规范
- **阶段三**：面向对象编程、命名空间、自动加载

### 不需要掌握

以下知识**不需要**提前掌握，会在学习过程中逐步学习：

- PSR 标准规范（从零开始学习）
- Composer 高级应用（从零开始学习）

### 如何检查

完成以下检查点，确认可以开始本阶段：

1. **基础检查**
   - [ ] 能够使用 Composer 管理依赖
   - [ ] 理解命名空间和自动加载
   - [ ] 能够编写规范的 PHP 代码

2. **理解检查**
   - [ ] 理解代码规范的重要性
   - [ ] 能够阅读和理解 PSR 标准
   - [ ] 理解 Composer 的基本概念

3. **实践检查**
   - [ ] 完成阶段三的所有练习
   - [ ] 能够使用 PSR-4 自动加载

**如果以上检查点都通过，可以开始阶段十二的学习。**

**如果某些检查点未通过，建议：**
- 复习阶段一、二、三的相关章节
- 完成前面阶段的练习
- 确保理解核心概念后再继续

## 学习时间估算

**建议学习时间：** 2-3 周

- 每天学习 1-2 小时
- 每周学习 5-6 天
- 作为补充学习资料，可随时查阅

**时间分配建议：**
- PSR 标准规范（12.1）：1-2 周
- Composer 高级应用（12.2）：1 周
- 阶段总结与复习：1 周

## 练习建议

### 基础练习（每章完成后）

每完成一章，建议完成 3-5 个小练习，例如：

1. **PSR 标准规范**
   - 按照 PSR-12 规范重构代码
   - 实现 PSR-4 自动加载
   - 使用 PSR-3 日志接口

2. **Composer 高级应用**
   - 编写和发布 Composer 包
   - 配置 Composer 脚本
   - 使用 Composer 插件

### 综合练习（阶段完成后）

3. **综合项目**
   - 在现有项目中应用 PSR 标准
   - 发布自己的 Composer 包
   - 建立团队代码规范

**项目要求：**
- 使用本阶段学到的所有知识点
- 代码规范、注释清晰
- 符合 PSR 标准
- 完善的文档说明
